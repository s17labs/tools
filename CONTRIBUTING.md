# Contributing to s17 Labs Tools

First off, thank you for considering contributing to s17 Labs! It's people like you that make the open-source community such a great place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
If you find a bug in any of the tools, please open an issue. Provide a clear title, a detailed description of the problem, and steps to reproduce it.

### Suggesting Enhancements
Have an idea for a new feature or a new tool? We are always looking for ways to expand the lab. Open an issue with the "enhancement" tag to start a discussion.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Ensure your code follows the existing style (clean, documented vanilla JS/HTML/CSS preferred).
3. Update the documentation (README) if you are adding a new tool.
4. Issue a pull request with a clear description of your changes.

## Development Principles
- **Privacy First**: Tools must run entirely on the client side. No data should be sent to external servers.
- **Lightweight**: Minimize external dependencies. Favor native Web APIs.
- **Midnight Aesthetic**: Maintain the established dark/accent theme for UI consistency.
